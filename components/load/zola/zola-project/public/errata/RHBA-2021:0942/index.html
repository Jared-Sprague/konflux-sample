RHBA-2021:0942 - Update to RHEL7-based OpenJDK S2I Images for OpenShift

Synopsis

Update to RHEL7-based OpenJDK S2I Images for OpenShift

Type/Severity

Bug Fix Advisory None

Topic

Updates are available for the RHEL7-based OpenJDK S2I images (OpenJDK versions 8 and 11) for OpenShift Container Platform 3.11 and 4.7.

Description

Red Hat JBoss Middleware for OpenShift provides images for many of the Red Hat Middleware products, for use with OpenShift Container Platform, with on-premise or private cloud deployments.

These images include fixes for CVE-2021-20264.

These images can be used with OpenShift Container Platform 3.11 and 4.

Solution

To update to the latest Red Hat JBoss Middleware for OpenShift Java S2I images run the following steps to pull in the content:

  1. On your master host(s), ensure you are logged into the CLI as a cluster administrator or user that has project administrator access to the global "openshift" project. For example:

$ oc login -u system:admin

  1. Run the following command to update the OpenShift Java S2I image streams in the "openshift" project:

$ oc replace --force -n openshift -f
https://raw.githubusercontent.com/jboss-container-images/openjdk/release/templates/image-streams.json

Note, this update of the image streams will pull images from registry.redhat.io, which requires authentication. Please refer to the following KB for details on how to configure your environment to use the new registry: https://access.redhat.com/RegistryAuthentication
