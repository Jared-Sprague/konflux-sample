How to force a kickstart installation to take place over a specific Ethernet device?

Solution Verified - Updated 7 Aug 2024

Environment

  • Red Hat Enterprise Linux 3, 4, 5, 6
  • kickstart

Issue

  • What option can be used to force a kickstart installation to take place over a specified Ethernet device?
  • How to use a specific Ethernet device for kickstart on system with multiple NIC cards.

Resolution

The ksdevice option at the install boot prompt ( or in PXE config ) can be used to setup prefered Ethernet device for kickstart :

boot: linux ks=kickstart.redhat.com:/redhat ksdevice=ethX

Replace "X" with preferred device. Example : 0, 1 .

The "ksdevice" takes one of 4 types of argument which tells the installation what network device to use for kickstart from network:

  • An argument like 'eth0' naming a specific interface

  • An argument like "00:12:34:56:78:9a" indicating the MAC address of a specific interface

  • The keyword `link' indicating that the first interface with link is up

  • The keyword `bootif' indicating that the MAC address indicated by the BOOTIF command line option will be used to locate the boot interface. BOOTIF is automagically supplied by pxelinux when the option `IPAPPEND 2' in the pxelinux.cfg file is included.

See /usr/share/doc/anaconda-*/command-line.txt for more information on ksdevice and other command line options for Anaconda.

Refer: Booting from the Network Using PXE
