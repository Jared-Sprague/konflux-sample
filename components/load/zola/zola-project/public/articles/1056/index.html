Why am I unable to boot from a Red Hat Enterprise Linux CD-ROM on an x86 system?

Updated 15 Sept 2012

Note: There are a few cases where the system BIOS does not allow the Red Hat Enterprise Linux CD-ROM to boot because of the size of the boot image on the CD-ROM itself. In cases such as these, a boot diskette should be made to boot the installation program. Once booted, the CD-ROMs work properly for the installation.

If you cannot boot from your Red Hat Enterprise Linux CD-ROM, you have two options:

  1. Change your BIOS so that booting from the CD-ROM is recognized first in the boot order

  2. Boot using a boot diskette you have created

To change your BIOS, refer to your system manual for the correct keyboard combination that allows you to access your BIOS, or you can read the key sequence needed while the system begins to boot (assuming it is displayed).

To create a boot diskette, follow the instructions in Section 2.5.3 Making an Installation Boot Diskette of the installation guide.

To boot Red Hat Enterprise Linux using a boot diskette, insert the diskette you have created into your diskette drive and then boot/reboot your computer. Make sure that your BIOS is set to use the diskette or removable diskette to boot.
