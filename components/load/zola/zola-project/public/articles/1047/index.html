How can I get my Red Hat system to obtain an IP address from my Windows Dynamic Host Configuration Protocol (DHCP) server?

Updated 15 Sept 2012

For some Dynamic Host Configuration Protocol (DHCP) servers it is necessary to use a different version of the DHCP client on your Red Hat system.

For Red Hat Enterprise Linux 2.1 users do the following:

first need to remove the default version:

     rpm -e dhcpcd

Next you will need to install the pump package. If your system is registered with the Red Hat Network (RHN), you can install the pump package using the following command:

     up2date pump

If you can not access Red Hat Network, then you should be able to install the pump RPM from your installation CDs.

After the RPM is installed, you will need to restart (or start) your networking service:

     service network restart
     service network start

Your system should now be able to obtain an IP address from your Windows DHCP server if your network interface is configured to obtain the IP address via DHCP.

Note: For Red Hat Enterprise Linux 3, there is no alternate supported DHCP packages but there is dhcpcd which is an alternative to the pump package and may be beneficial.

Please visit DHCPCD Homepage for more information.
